Overview

In this game the player is tasked with visually identify a colour based in its RGB value. RGB colours are created by mixing differing levels of red, green and blue. Each red, green and blue component can have a value ranging from 0 to 255. For example, a fully green colour would have the RGB values of (0,255,0). The player is able to choose between ‘easy’ mode and ‘hard’ mode. In easy mode the player has 3 colours to choose between. In hard mode, there are six colours.
A combination of ‘Math.random()’ and Math.floor() is used to generate individual RGB values and create one RGB colour. This done using a ‘for loop’ either 3 or 6 times depending on the difficulty selected. The ‘.push()’ method is used to put each new colour in the ‘colours’ array. Another random number is generated, and this is used to select the number from the ‘colours’ array with the same index. This is assigned as the ‘pickedColour’, the colour the player will be attempting to guess. The RGB values of this colour is displayed to the player using the ‘.textContent’ method.
A ‘for loop’ is used to assign each square with a background colour form the ‘colours’ array. if there are more squares than colours (i.e. in easy mode), the additional squares are hidden. An event listener is added to each square so that when clicked, a function checks if the background colour of that square is the same value as the ‘pickedColour’. If so, the player has guessed correctly and won. If not, the squares background coloured is changed to that of the pages background, making it disappear.
Whenever the game is reset or the difficulty is changed, a new array of colours is generated and a new ‘pickedColour’ is selected.
